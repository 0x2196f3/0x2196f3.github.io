OpenWrt设置usb扩容

## OpenWrt设置usb扩容

(原文)[https://openwrt.org/docs/guide-user/additional-software/extroot_configuration]

# 连接SSH

# 安装软件包

opkg update
opkg install block-mount kmod-fs-ext4 e2fsprogs parted kmod-usb-storage

# 查看USB设备位置

ls -l /sys/block

如果是无线路由器只有一个外接存储，大概率是sda

lrwxrwxrwx    1 root     root             0 xxx xx xx:xx sda -> ../devices/xxx/block/sda

# 在USB上创建分区，格式化，设置exroot并自动挂载

#Partitioning and formatting
DISK="/dev/sda"
parted -s ${DISK} -- mklabel gpt mkpart extroot 2048s -2048s
DEVICE="${DISK}1"
mkfs.ext4 -L extroot ${DEVICE}

#Configuring extroot
eval $(block info ${DEVICE} | grep -o -e 'UUID="\S*"')
eval $(block info | grep -o -e 'MOUNT="\S*/overlay"')
uci -q delete fstab.extroot
uci set fstab.extroot="mount"
uci set fstab.extroot.uuid="${UUID}"
uci set fstab.extroot.target="${MOUNT}"
uci commit fstab

#Configuring rootfs_data / ubifs
ORIG="$(block info | sed -n -e '/MOUNT="\S*\/overlay"/s/:\s.*$//p')"
uci -q delete fstab.rwm
uci set fstab.rwm="mount"
uci set fstab.rwm.device="${ORIG}"
uci set fstab.rwm.target="/rwm"
uci commit fstab

docker部署llama-swap

gpt-oss刚发布，ollama的支持有问题，性能和llama-cpp相比有几倍的差距，但是llama-cpp一个进程只能加载一个gguf模型，api也比较简陋，无法长期使用

不过ollama性能问题估计过几天就修复了

# Ollama Llama-cpp 性能对比

gpt-oss:20btokens/s
llama-cpp (vulkan)96.52
llama-cpp (rocm on Windows)94.11
ollama (rocm)35.72

# llama-swap

llama-server和llama-swap打包进同一个docker，通过启动多个llama-server进程切换gguf模型，提供一个简单的webui，可以手动加载/卸载模型，也可以在网页上对话

由于llama-cpp没有提供Linux的ROCM二进制文件，只能使用Vulkan，单卡性能和ROCM基本没有区别

Vulkan可以在多张不同品牌的GPU上运行同一个模型，实测一张AMD独显+一张Intel独显能运行，性能可以接受 (16GB+16GB，qwq:32b Q4_K_M 15 token/s)

ZFS Dataset和Directory冲突

ZFS Dataset下已有 ./a 目录，zfs create pool/a 会把数据集挂载到 ./a，原目录及其内容被隐藏（不删除）。

恢复原目录

sudo zfs unmount pool/a

寻找被 ZFS 挂载而隐藏的目录

# pwd是一个ZFS Pool/Dataset 挂载点
sudo find . -type d -print0 | xargs -0 stat -c "%d %n" | sort -n | awk 'NR==1{p=$1;next} $1!=p{print "ZFS mount at", $2} {p=$1}'

ZFS 强制删除dataset

TrueNAS删除dataset时经常遇到dataset is busy, -f没用也不说是什么进程占用这个dataset,解决方法是先改名再删除

rename dataset

sudo zfs rename pool/olddataset pool/newdataset

destroy dataset

sudo zfs destroy -R pool/newdataset

Fedora+Windows双系统启动配置

# 1. Fedora 自动删除旧内核

sudo nano /etc/dnf/dnf.conf

保留最新的2个内核

[main]
installonly_limit=2

# 2. GRUB2 设置默认启动分区

sudo nano /etc/default/grub

三种模式

  • 记忆上次选择的启动项
GRUB_DEFAULT=saved
GRUB_SAVEDEFAULT=true
  • 选择第3个启动项
GRUB_DEFAULT=2
  • 选择该名称的启动项
GRUB_DEFAULT="Windows Boot Manager (on /dev/sda1)"

保存

sudo grub2-mkconfig -o /boot/grub2/grub.cfg

搬瓦工VPS配置自定义IPv6地址

搬瓦工提供的ipv6是通过6in4隧道连接到另一台服务器的ipv6出口的,公网ipv6地址在ipv6net上,不是eth0

  • ipv6直连中国速度比ipv4慢不少
  • 删了ipv6重新分配还是原来的那个
  • 可以先用ipv6测试协议会不会被墙，没事再换成ipv4

搬瓦工后台通过ipv6net这个service自动配置6in4隧道,配置脚本在/etc/init_ipv6_tunnel,但修改这个脚本重启会复原
需要禁用ipv6net这个服务,再手动配置/etc/network/interfaces

禁用ipv6net服务

systemctl stop ipv6net
systemctl disable ipv6net
reboot

手动配置ipv6地址(Ubuntu/Debian)

nano /etc/network/interfaces

添加如下

# 所有ipv6地址都不带中括号'['和']'
auto ipv6net
iface ipv6net inet6 v4tunnel
        address [your custom ipv6]
        netmask 64
        endpoint *.*.*.*
        local *.*.*.*
        ttl 255
        gateway [ipv6 gateway]

        up   /sbin/ip -6 addr add [another custom ipv6]/64 dev $IFACE
        up   /sbin/ip -6 addr add [yet another custom ipv6]/64 dev $IFACE

        down /sbin/ip -6 addr del [another custom ipv6]/64 dev $IFACE
        down /sbin/ip -6 addr del [yet another custom ipv6]/64 dev $IFACE

ip地址根据搬瓦工后台的配置照抄,up和down的ipv6地址一一对应,复制多行可以添加多个自定义ipv6地址
